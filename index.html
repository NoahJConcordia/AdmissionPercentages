By default, content division elements (divs) have no padding, margin or border. They extend horizontally as wide as possible but tightly conform to the top and bottom of whatever content's inside.

Heading and paragraph elements have top and bottom margins by default that add vertical distance from other HTML elements.

We're primarily concerned right now with vertical layouts — getting our HTML content to stack nicely on desktop and mobile screens.

Setting a max-width is helpful for this. It helps keep text, images and data visualizations from taking up the entire width of a large screen. You provide a maximum width that an element, like a div, can't exceed. But it can get narrower as the screen shrinks.

In the CSS file, give this div a max-width of 700px.

4

How can we get this div get horizontally centered?

Use the same max-width attribute as the div above, but set the left and right margins to "auto", which means they're calculated automatically based on the width of the browser window.

This allows the div to hang out in the middle. As the screen gets narrower, the margins will eventually shrink away to nothing and it'll take up the full width.

(This is similar to how your recipe text was centered on the webpage in an earlier in-class activity, if you go back to peek at the CSS).
